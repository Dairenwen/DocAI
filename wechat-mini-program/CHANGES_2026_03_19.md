## 修复内容

### 1. API 端点修复

#### 批量删除文档端点 ✓ 已修复
- **文件**: [miniprogram/api/docai.js](miniprogram/api/docai.js#L119)
- **问题**: 使用了错误的 HTTP 方法和端点路径
  - 错误: `request({ url: '/source/batch', method: 'DELETE' })`
  - 正确: `request({ url: '/source/batch-delete', method: 'POST' })`
- **原因**: 后端端点定义为 `@PostMapping("/batch-delete")`

## 验证结果

### 已验证正确的接口

#### 用户认证 ✓
- `POST /users/auth` - 统一登录/注册
- `GET /users/info` - 获取用户信息
- `POST /users/logout` - 登出

#### 源文档管理 ✓
- `POST /source/upload` - 上传文档
- `DELETE /source/{docId}` - 删除单个文档
- `POST /source/batch-delete` - 批量删除文档 **[修复]**
- `GET /source/documents` - 获取文档列表
- `GET /source/{docId}` - 获取单个文档

#### AI 对话 ✓
- `POST /ai/chat/stream` (SSE) - AI 流式对话
- 参数映射: `message`/`documentId` → `userInput`/`fileId`

#### 模板自动填表 ✓
- `POST /template/upload` - 上传模板
- `POST /template/{id}/parse` - 解析槽位
- `POST /template/{id}/fill` - 自动填表
- `GET /template/list` - 列表模板
- `GET /template/{id}/audit` - 审计日志
- `GET /template/{id}/decisions` - 决策记录

### 响应格式验证 ✓

所有接口遵循统一的响应格式：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { /* 实际数据 */ }
}
```

前端正确处理了：
- ✓ `code !== 200` 自动拒绝
- ✓ `statusCode === 401` 自动清除 token
- ✓ 数据属于 `data` 字段的嵌套结构

## 建议验证项

### 1. 时间戳格式兼容性
- 后端: Java LocalDateTime (ISO 8601 格式)
- 前端: JavaScript Date 需要手动测试
- **建议**: 在真机环境测试 formatDate 函数输出

### 2. SSE 流式响应
- 小程序不原生支持 SSE，当前使用 wx.request 等待完整响应
- **建议**: 测试大数据量的 AI 对话响应是否会被截断

### 3. 文件上传超时
- 配置: 300 秒 (5 分钟)
- **建议**: 根据实际场景验证是否充足
