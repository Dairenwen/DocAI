# 小程序内容安全接入点与分工建议

更新时间：2026-04-05

## 目标

在不大改当前主业务链路的前提下，为小程序补齐上线前必需的内容安全接入思路，覆盖：

- 文件上传
- AI 输入
- AI 输出
- 会话内容

## 当前接入点梳理

### 1. 文件上传

当前前端入口：

- `miniprogram/pages/docai/documents/index.js`
  - 文档页上传资料
- `miniprogram/pages/docai/autofill/index.js`
  - 智能填表页上传数据源
  - 智能填表页上传模板
- `miniprogram/api/docai.js`
  - `uploadDocument(filePath, fileName)`
  - `uploadTemplateFile(filePath, fileName)`

建议前端处理：

- 继续保留格式、大小、数量校验。
- 上传前提示仅支持业务资料文件，禁止上传涉黄、涉政、暴恐、侵权、木马或无关内容。
- 上传失败时只给用户可读提示，不透出内部服务细节。
- 若后端返回内容安全拦截错误码，前端统一提示“文件内容暂不支持处理，请更换文件后重试”。

建议后端处理：

- 校验扩展名、MIME、文件大小、单次数量。
- 接入文件病毒扫描或恶意文件检测。
- 对文件文本内容做内容安全检测。
- 对 OCR 文本、解析文本也做二次检测，避免绕过。
- 对命中风险的文件禁止入库或标记为隔离状态。
- 返回明确错误码，例如：
  - `FILE_TYPE_NOT_ALLOWED`
  - `FILE_TOO_LARGE`
  - `FILE_SECURITY_REJECTED`
  - `FILE_PARSE_FAILED`

### 2. AI 输入

当前前端入口：

- `miniprogram/pages/docai/chat/index.js`
  - 用户输入问答内容
- `miniprogram/pages/docai/autofill/index.js`
  - 用户填写“我的需求”
- `miniprogram/api/docai.js`
  - `aiChat(data, options)`
  - `fillTemplate(templateId, docIds, userRequirement)`

建议前端处理：

- 限制输入长度、去掉纯空白输入。
- 发送前增加简单风险提示，告知请勿输入违法违规、敏感或无关内容。
- 若后端返回命中安全策略，前端提示“输入内容暂不支持处理，请调整后重试”。

建议后端处理：

- 对聊天输入、模板填表需求统一做文本安全检测。
- 对命中风险的输入禁止进入模型调用。
- 对命中记录留审计日志，包含用户、时间、场景、命中策略。
- 对高风险输入直接拒绝；对中风险输入可做人工复核或脱敏后再处理。

### 3. AI 输出

当前前端入口：

- `miniprogram/pages/docai/chat/index.js`
  - 展示 AI 回复
- `miniprogram/pages/docai/autofill/index.js`
  - 展示填表摘要、决策预览、结果文案
- `miniprogram/api/docai.js`
  - `doAiChatRequest(...)`
  - `fillTemplate(...)`

建议前端处理：

- 前端只展示后端已确认可下发的内容，不自行判断是否合规。
- 若后端返回“输出被拦截”状态，前端统一提示“本次结果暂无法展示，请稍后重试”。
- 不要将命中安全策略的原始输出继续缓存到本地。

建议后端处理：

- 对模型输出做文本安全检测。
- 对结构化字段、摘要、决策理由都做检测，不只检测整段文本。
- 对命中风险的输出进行拦截、替换或降级。
- 下载文件生成前，对落盘内容再做一次终检。
- 保留输出审核日志，支持复核与追踪。

### 4. 会话内容

当前前端入口：

- `miniprogram/pages/docai/chat/index.js`
  - 会话列表、消息列表、当前会话切换
- `miniprogram/utils/storage.js`
  - 当前账号下会话缓存与当前会话索引

建议前端处理：

- 继续按 `userId` 隔离本地缓存。
- 对命中安全策略的会话消息，不再继续写入本地缓存。
- 会话删除、退出登录、切换账号时清理临时会话态。

建议后端处理：

- 入库前校验用户输入与模型输出。
- 对命中风险的消息做状态标记，必要时禁止持久化。
- 增加会话级审计字段，例如风险等级、拦截原因、处理时间。

## 推荐前后端分工

### 前端负责

- 做轻量规则校验：
  - 文件格式
  - 文件大小
  - 单次数量
  - 输入长度
- 提供明确用户提示：
  - 上传失败
  - 内容不支持
  - 结果暂不可展示
- 不缓存被拦截内容。
- 展示合规入口：
  - 隐私政策
  - 用户协议
  - 联系我们
  - 账号注销

### 后端负责

- 真正的内容安全判定和拦截。
- 文件扫描、文本审核、模型输出审核。
- 审计日志、风险状态、错误码设计。
- 对下载文件和持久化内容做终检。

## 最小落地顺序

1. 后端先补统一错误码和安全拦截响应。
2. 文件上传接口补 MIME / 扩展名 / 大小 / 数量 / 安全检测。
3. AI 输入与 AI 输出统一接安全检测。
4. 前端统一映射安全类错误提示。
5. 最后补审计日志和复核后台。

## 提审前至少要准备的后台能力

- 文件安全校验
- 文本内容安全校验
- AI 输出安全校验
- 命中策略后的错误码与提示语
- 审计日志与问题追踪
