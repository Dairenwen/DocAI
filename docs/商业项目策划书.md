# DocAI 智能文档处理系统 - 商业项目策划书

## 一、项目概述

### 1.1 项目名称
**DocAI —— 基于大模型的智能文档信息提取与自动填表系统**

### 1.2 项目定位
DocAI 是一款面向企事业单位、科研院校、政府机关等组织的**智能文档自动化处理平台**。系统通过引入大语言模型（LLM）与多格式文档解析技术，实现从**非结构化文档中自动提取关键信息**，并**智能填充到用户提供的模板表格中**，大幅降低人工录入成本，提升办公效率和数据准确性。

在数字化转型的大背景下，各行各业每天产生海量的文档数据——项目申报书、科研报告、合同文件、政务公文、病历档案等，这些文档中蕴含着大量结构化信息，却长期依赖人工手动阅读、提取和录入。DocAI 正是为解决这一普遍性痛点而诞生的产品。

### 1.3 核心价值主张

| 价值维度 | 传统方式 | DocAI 方案 | 提升幅度 |
|---------|---------|-----------|---------|
| **处理效率** | 数小时人工整理 | 90秒内自动完成 | **10~50倍** |
| **准确率** | 人工疲劳导致5~15%错误率 | AI+规则混合决策 ≥80% | **显著降低错误** |
| **格式支持** | 每种格式需要不同工具 | docx/xlsx/txt/md 统一处理 | **一站式覆盖** |
| **可追溯性** | 无法追溯数据来源 | 每个填入值均可溯源 | **全链路审计** |
| **人力成本** | 需要专人负责数据录入 | 自动化释放人力 | **降低60%+** |
| **一致性** | 不同人员录入标准不统一 | 系统统一标准化处理 | **100%一致** |

### 1.4 项目愿景

**短期目标（1年内）**：成为国内领先的AI文档信息提取与自动填表工具，服务1000+用户
**中期目标（1-3年）**：打造企业文档智能中枢平台，接入OA/ERP/CRM等业务系统
**长期目标（3-5年）**：构建行业级文档智能处理生态，覆盖政务、教育、金融、医疗四大垂直领域

## 二、市场分析

### 2.1 行业背景与市场规模

**全球智能文档处理市场**：
- 2024年全球智能文档处理（IDP）市场规模约为 **42亿美元**
- 预计2030年将达到 **175亿美元**，年复合增长率（CAGR）约 **26.8%**
- 中国市场在全球占比约15%，增速高于全球平均水平

**驱动因素**：
1. **数字化转型加速**：企业/政府大力推进无纸化办公，但文档数字化远不等于"信息数字化"
2. **AI技术成熟**：大语言模型（LLM）的语义理解能力使非结构化文档自动处理成为可能
3. **劳动力成本上升**：数据录入岗位人力成本持续增长，自动化需求强烈
4. **合规性要求**：金融、医疗等行业对数据填写的准确性和可追溯性要求越来越高

### 2.4 核心竞争优势（护城河）

1. **规则+AI混合决策架构**：90%字段纯规则匹配（快+稳），仅难例调用LLM（准+智能），独创的五级候选召回策略确保匹配精度
2. **全链路可审计**：每个填入值可追溯到来源文档的具体段落，满足合规审计需求
3. **多格式统一框架**：docx/xlsx/txt/md 四种格式统一提取管线，新增格式只需实现解析适配器
4. **预提取+实时匹配**：源文档上传时异步预提取，模板填充时毫秒级匹配，双阶段架构确保响应速度
5. **开源可私有化**：支持完全私有化部署，数据不出企业内网，适合政务/金融等敏感行业
6. **微服务可横向扩展**：基于 Spring Cloud 微服务架构，ai-service 可多实例水平扩展
7. **低成本AI调用**：通过规则优先策略，LLM调用量降低80%以上，大幅降低API使用成本

## 三、产品功能

### 3.1 三大核心模块

#### 模块一：文档智能操作交互
- **自然语言对话式文档操作**：用户通过文字指令指挥AI分析、编辑、润色文档
- **SSE实时流式响应**：AI回复逐字显示，类似ChatGPT交互体验
- **AI文档在线编辑**：用户说"删除第三段"，AI自动生成修改后文档并提供下载
- **多模型热切换**：支持通义千问（qwen-turbo/plus/max）和 DeepSeek 模型实时切换
- **对话历史持久化**：对话记录保存到数据库，支持跨设备访问
- **智能意图识别**：自动区分查询类/编辑类/图表类请求，路由到不同处理流程
- **9种快捷命令**：内容摘要、信息提取、润色优化、格式调整、数据分析、删除冗余、内容补充、翻译文档、导出结果
- **多格式导出**：AI结果可导出为 Word(.docx)、纯文本(.txt)、Markdown(.md)

#### 模块二：非结构化文档信息提取
- **四格式全覆盖**：支持 docx/xlsx/txt/md 四种主流办公文档格式
- **12大类实体提取**：项目信息、人员信息、机构信息、联系方式、证件信息、时间信息、经费信息、研究信息、地理信息、数量统计、非结构化实体、表格数据
- **单文档独立抽取**：一文档一调用LLM，避免跨文档信息污染
- **字段自动标准化**：40+中文别名到30+标准key的自动映射
- **docx智能表格识别**：自动检测key-value行结构和多行数据表格
- **Markdown表格增强解析**：支持标准Markdown表格（含分隔行），自动查找表头
- **多值完整提取**：同类字段出现多次时逐一独立提取（3个国家→3条记录）
- **规则引擎兜底**：LLM不可用时自动降级到增强版规则引擎（"键:值"匹配+实体识别）
- **异步预提取架构**：上传后立即返回，后台异步提取，不阻塞用户操作

#### 模块三：表格自定义数据填写
- **双模板支持**：支持 Word(.docx) 和 Excel(.xlsx) 模板文件
- **三类槽位自动识别**：左标签右空白、上标签下空白、内联标签（标签:____）
- **八阶段处理流水线**：数据契约→提取→标准化→槽位识别→候选召回→难例判定→置信度融合→写回
- **五级候选召回策略**：别名匹配(0.34)+类型匹配(0.18)+多文档投票(0.16)+上下文(0.10)+置信度(0.08)+需求匹配(0.14)
- **值类型交叉验证**：数字/日期/电话/人员槽位自动拒绝不兼容候选值
- **短标签精确匹配**：≤2字符标签禁用模糊匹配，防止跨字段错填
- **复合标签拆分**：含"/"的标签（如"国家/地区"）自动拆分分别匹配，分行写入
- **directTableCopy快速路径**：表头型模板自动检测，从源Excel直接复制数据行（38000+行→4500+行，5秒完成）
- **统计聚合引擎**：自动识别SUM/AVG/MAX/MIN/COUNT类汇总槽位，从源数据计算统计值
- **需求驱动精准填充**：用户可输入自然语言需求（时间/地点/类型），系统优先填入符合条件的数据
- **多源文档合并填充**：支持多个不同类型源文档（docx+xlsx+txt+md混合）共同填充同一模板
- **多阶段表头引导提取**：docx表格直接提取→正则全量匹配→LLM上下文精准提取
- **实时在线预览**：基于VueOffice插件支持 .docx/.xlsx 填充结果在线预览
- **全链路审计日志**：每个字段记录决策方式、来源文档、置信度、候选列表

### 3.2 辅助功能
- **用户认证与权限管理**：JWT Token 认证（20小时有效期），支持密码/邮箱验证码双模式登录
- **文件云存储**：支持阿里云OSS / 本地存储，可配置切换
- **AI对话历史管理**：对话列表、消息管理、会话置顶、关联文档
- **LLM供应商动态切换**：DashScope（8个模型） / DeepSeek（2个模型），实时切换
- **邮件服务**：验证码发送、AI结果邮件发送、填表结果邮件发送
- **文档批量操作**：批量下载、批量删除
- **智能轮询机制**：自适应间隔轮询（前30秒3秒/次，之后8秒/次），高效获取提取状态
