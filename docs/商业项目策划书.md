# DocAI 智能文档处理系统 - 商业项目策划书

## 一、项目概述

### 1.1 项目名称
**DocAI —— 基于大模型的智能文档信息提取与自动填表系统**

### 1.2 项目定位
DocAI 是一款面向企事业单位、科研院校、政府机关等组织的**智能文档自动化处理平台**。系统通过引入大语言模型（LLM）与多格式文档解析技术，实现从**非结构化文档中自动提取关键信息**，并**智能填充到用户提供的模板表格中**，大幅降低人工录入成本，提升办公效率和数据准确性。

### 1.3 核心价值主张
- **效率提升**：将传统需要数小时的手工信息整理工作缩短至90秒以内完成
- **准确率保障**：通过"规则召回 + LLM难例判定 + 拒填策略"三级保障机制，确保填表准确率 ≥ 80%
- **全格式支持**：支持 docx、xlsx、txt、md 四种主流文档格式的信息提取
- **可追溯审计**：每个填入值均可追溯到来源文档与原文证据

## 二、市场分析

### 2.1 目标市场
| 行业领域 | 典型场景 | 市场规模 |
|---------|---------|---------|
| 教育科研 | 项目申报书信息提取、科研成果汇总 | 全国高校3000+所 |
| 政府机关 | 公文信息整理、表格自动填写 | 各级政府机构 |
| 企业办公 | 合同信息提取、报表汇总 | 中大型企业 |
| 金融保险 | 理赔单据信息识别、客户资料整理 | 金融机构 |

### 2.2 竞品分析
| 竞品 | 优势 | 劣势 |
|------|------|------|
| 传统OCR工具 | 图像识别成熟 | 不支持语义理解和智能填表 |
| 通用AI助手 | 对话能力强 | 无法直接操作文档文件 |
| RPA工具 | 流程自动化成熟 | 需要预设规则，灵活性差 |
| **DocAI** | **语义理解+自动填表+可追溯** | **需要网络连接LLM服务** |

### 2.3 核心竞争优势
1. **端到端自动化**：从文档上传到结果下载，全流程自动完成
2. **双模型协同架构**：Qwen-Doc-Turbo专注抽取 + qwen3.5-plus判定难例
3. **规则+AI混合策略**：优先规则匹配，仅难例调用LLM，兼顾速度和准确率
4. **全链路可审计**：支持填写结果追溯到来源文档的具体段落

## 三、产品功能

### 3.1 三大核心模块

#### 模块一：文档智能操作交互
- 自然语言对话式文档操作
- 支持文档摘要、内容提取、格式优化
- SSE实时流式响应

#### 模块二：非结构化文档信息提取
- 支持 docx/xlsx/txt/md 四种格式
- 单文档独立抽取，避免跨文档污染
- 字段自动标准化与别名词典构建
- 字段类型自动识别（text/date/number/phone/org/person/enum）

#### 模块三：表格自定义数据填写
- 支持 docx 和 xlsx 模板
- 三类槽位自动识别（左标签右空白、上标签下空白、内联标签）
- 五级候选召回策略
- 置信度分级决策（≥0.85直写、0.70~0.85复核、<0.70拒填）
- 写回后二次校验

### 3.2 辅助功能
- 用户认证与权限管理（JWT Token）
- 文件云存储（支持阿里云OSS / 本地存储）
- AI对话历史管理
- LLM供应商切换（DashScope / DeepSeek）
- 邮件发送（验证码 / 附件）
