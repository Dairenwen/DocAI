# DocAI 本地模型部署指南 — MacBook Pro M5 Max

> 本文档介绍如何在 MacBook Pro（M5 Max 芯片）上本地部署 Qwen2.5-VL-7B-Instruct 多模态视觉语言模型，并与 Windows 主机上的 DocAI 系统联调。

---

## 一、方案概述

### 1.1 部署拓扑

```
┌─────────────────────────────────────────┐
│         Windows 主机 (DocAI 系统)         │
│  ┌─────────────┐  ┌──────────────────┐  │
│  │ 前端 + Nginx │  │ Gateway + 微服务  │  │
│  │  (8080)      │  │ (18080/9001-9003)│  │
│  └──────────────┘  └────────┬─────────┘  │
│                             │             │
│        ai-service 通过 HTTP 调用          │
│        http://<mac-ip>:8000/v1            │
└─────────────────────┬───────────────────┘
                      │ 局域网 / 内网
┌─────────────────────▼───────────────────┐
│     MacBook Pro M5 Max (模型推理服务器)    │
│  ┌──────────────────────────────────┐    │
│  │  Ollama / vLLM 推理引擎            │    │
│  │  模型: qwen2.5-vl-7b-instruct    │    │
│  │  端口: 8000 (OpenAI 兼容 API)     │    │
│  │  内存: 统一内存 ≥ 36GB            │    │
│  └──────────────────────────────────┘    │
└──────────────────────────────────────────┘
```

### 1.2 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 芯片 | Apple M3 Pro 以上 | Apple M5 Max |
| 统一内存 | 36GB | 48GB+ |
| 磁盘空间 | 30GB 可用 | 50GB+ |
| macOS 版本 | macOS 14 Sonoma | macOS 15+ |

> **关于 Apple Silicon 统一内存**：M5 Max 的 48GB/64GB 统一内存由 CPU 和 GPU 共享，7B 参数量的模型约需 14-16GB 显存，统一内存架构非常适合本地推理。

### 1.3 方案选择

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| **Ollama** | 安装极简、自动管理模型、原生支持 Apple Silicon | 模型更新依赖 Ollama 发布 | 快速上手、个人开发 |
| **vLLM** | 高吞吐、支持连续批处理、更灵活 | 安装配置较复杂、macOS 支持需 Metal 后端 | 生产环境、高并发 |
| **llama.cpp (推荐)** | 原生 Metal 加速、轻量、GGUF 量化模型 | 需手动下载模型 | 内存有限、最佳性能 |

## 二、方案一：Ollama 部署（推荐）

### 2.1 安装 Ollama

```bash
# 下载安装 Ollama (macOS 原生支持 Apple Silicon)
curl -fsSL https://ollama.com/install.sh | sh

# 验证安装
ollama --version
```

或从 [Ollama 官网](https://ollama.com/download/mac) 下载 .dmg 安装包。

### 2.2 下载并运行 Qwen2.5-VL 模型

```bash
# 拉取 Qwen2.5-VL-7B 模型 (约 4.7GB GGUF 量化版)
ollama pull qwen2.5vl:7b

# 验证模型可用
ollama list

# 启动模型服务 (默认端口 11434)
ollama serve
```

> **注意**：Ollama 默认监听 `127.0.0.1:11434`，需要修改为 `0.0.0.0` 以允许远程访问。

### 2.3 配置远程访问

```bash
# 设置环境变量允许所有网络接口访问
export OLLAMA_HOST=0.0.0.0:11434

# 如果使用 launchd 管理 Ollama 服务，修改 plist：
# 编辑 ~/Library/LaunchAgents/com.ollama.server.plist
# 在 <dict> 内添加：
# <key>EnvironmentVariables</key>
# <dict>
#   <key>OLLAMA_HOST</key>
#   <string>0.0.0.0:11434</string>
# </dict>

# 重启 Ollama 服务
ollama serve
```

### 2.4 验证 API 可用

```bash
# 在 MacBook 本地测试
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5vl:7b",
    "messages": [
      {
        "role": "user",
        "content": "你好，请介绍一下你自己。"
      }
    ]
  }'

# 从 Windows 主机测试（替换 <mac-ip> 为 MacBook 的局域网 IP）
# PowerShell:
Invoke-RestMethod -Method Post `
  -Uri "http://<mac-ip>:11434/v1/chat/completions" `
  -ContentType "application/json" `
  -Body '{"model":"qwen2.5vl:7b","messages":[{"role":"user","content":"Hello"}]}'
```

### 2.5 获取 MacBook 局域网 IP

```bash
# 在 MacBook 终端执行
ifconfig en0 | grep "inet " | awk '{print $2}'
# 示例输出: 192.168.1.100
```

## 三、方案二：llama.cpp + GGUF 量化部署

### 3.1 安装 llama.cpp

```bash
# 安装 Homebrew (如未安装)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 编译 llama.cpp (自动启用 Metal 加速)
git clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp
make LLAMA_METAL=1 -j$(sysctl -n hw.ncpu)
```

### 3.2 下载量化模型

```bash
# 使用 Hugging Face CLI 下载 GGUF 量化模型
pip3 install huggingface-hub
huggingface-cli download Qwen/Qwen2.5-VL-7B-Instruct-GGUF \
  qwen2.5-vl-7b-instruct-q4_k_m.gguf \
  --local-dir ./models

# 或使用镜像站下载 (国内加速)
# HF_ENDPOINT=https://hf-mirror.com huggingface-cli download ...
```

### 3.3 启动 OpenAI 兼容 API 服务

```bash
# 启动服务器 (端口 8000, 绑定所有网络接口)
./llama-server \
  --model ./models/qwen2.5-vl-7b-instruct-q4_k_m.gguf \
  --host 0.0.0.0 \
  --port 8000 \
  --n-gpu-layers 99 \
  --ctx-size 8192 \
  --threads $(sysctl -n hw.ncpu)
```

> `--n-gpu-layers 99` 表示尽可能多地将模型层加载到 GPU（Metal），M5 Max 的统一内存可完全加载 7B 模型。

## 四、方案三：vLLM 部署

> **注意**：截至 2026 年初，vLLM 对 macOS Apple Silicon 的支持仍在发展中。如果遇到兼容性问题，建议优先使用 Ollama 或 llama.cpp。

### 4.1 安装 vLLM

```bash
# 创建 Python 虚拟环境
python3 -m venv vllm-env
source vllm-env/bin/activate

# 安装 vLLM (检查是否支持 macOS Metal)
pip install vllm

# 如果 pip 安装不支持 macOS，尝试从源码编译：
# git clone https://github.com/vllm-project/vllm.git
# cd vllm
# pip install -e .
```

### 4.2 下载模型

```bash
# 从 Hugging Face 下载完整模型权重
huggingface-cli download Qwen/Qwen2.5-VL-7B-Instruct \
  --local-dir ./models/Qwen2.5-VL-7B-Instruct
```

### 4.3 启动 OpenAI 兼容 API 服务

```bash
vllm serve Qwen/Qwen2.5-VL-7B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --max-model-len 8192 \
  --trust-remote-code
```

## 五、DocAI 项目集成配置

### 5.1 方案 A：修改 DashScopeProvider 连接本地模型

当本地模型部署完成后，需要让 DocAI 的 ai-service 连接 MacBook 上的推理服务。

**修改 `DashScopeProvider.java` 中的 VL_BASE_URL**：

```java
// 原配置（连接阿里云 DashScope 云端 API）
private static final String VL_BASE_URL = 
    "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions";

// 改为本地模型地址（替换 <mac-ip> 为 MacBook 局域网 IP）
// Ollama 方案：
private static final String VL_BASE_URL = 
    "http://192.168.1.100:11434/v1/chat/completions";

// llama.cpp 方案：
private static final String VL_BASE_URL = 
    "http://192.168.1.100:8000/v1/chat/completions";
```

### 5.2 方案 B：通过环境变量配置（推荐）

为避免硬编码 IP 地址，建议通过环境变量控制：

**Windows PowerShell 设置**：
```powershell
# 设置本地模型 API 地址
$env:DOC_VL_BASE_URL = "http://192.168.1.100:11434/v1/chat/completions"

# Ollama 不需要 API Key，但 DashScope Provider 会校验
# 设置一个任意非空值即可
$env:DOC_DASHSCOPE_API_KEY = "ollama-local"
```

**修改 DashScopeProvider.java 支持环境变量**（可选）：
```java
private static final String VL_BASE_URL = 
    System.getenv().getOrDefault("DOC_VL_BASE_URL",
        "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions");
```

### 5.3 Ollama 模型名称映射

Ollama 中的模型名称与 DashScope 不同，需要在代码中处理映射：

| DashScope 模型名 | Ollama 模型名 | 说明 |
|-----------------|--------------|------|
| qwen2.5-vl-7b-instruct | qwen2.5vl:7b | 提取用 VL 模型 |
| qwen-plus | qwen2.5:14b | 难例判定/对话（可选） |
| qwen-turbo | qwen2.5:7b | 对话（可选） |

如果使用 Ollama，可在 `DashScopeProvider.java` 的 `generateMultiModalText` 方法中替换 `modelName`：

```java
// 本地 Ollama 模型名称映射
if ("qwen2.5-vl-7b-instruct".equals(modelName)) {
    modelName = "qwen2.5vl:7b";
}
```

### 5.4 网络连通性测试

在 Windows 主机上验证与 MacBook 的连接：

```powershell
# 1. Ping 测试
ping 192.168.1.100

# 2. 端口测试
Test-NetConnection -ComputerName 192.168.1.100 -Port 11434

# 3. API 调用测试
$body = @{
    model = "qwen2.5vl:7b"
    messages = @(
        @{
            role = "user"
            content = "Hello, are you working?"
        }
    )
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Method Post `
  -Uri "http://192.168.1.100:11434/v1/chat/completions" `
  -ContentType "application/json" `
  -Body $body
```

## 六、macOS 防火墙配置

如果 Windows 无法连接 MacBook 上的模型服务，检查防火墙设置：

```bash
# 查看防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 如果防火墙已开启，添加例外
# 方法1：系统偏好设置 → 安全性与隐私 → 防火墙 → 防火墙选项 → 添加应用
# 方法2：命令行
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/ollama
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp /usr/local/bin/ollama
```

## 七、性能优化

### 7.1 Ollama 性能调优

```bash
# 设置 GPU 层数（M5 Max 建议全部加载）
export OLLAMA_NUM_GPU=99

# 设置上下文长度（根据文档长度调整）
export OLLAMA_NUM_CTX=8192

# 设置并发请求数
export OLLAMA_NUM_PARALLEL=2

# 设置模型常驻内存（避免反复加载）
export OLLAMA_KEEP_ALIVE=24h
```

### 7.2 推理速度参考

| 配置 | 首次响应(TTFT) | 生成速度 | 说明 |
|------|--------------|---------|------|
| M5 Max 48GB | ~1-2s | ~30-40 tokens/s | Q4 量化，全 GPU 加载 |
| M5 Max 64GB | ~0.8-1.5s | ~35-45 tokens/s | Q4 量化，全 GPU 加载 |
| M3 Pro 36GB | ~2-4s | ~15-25 tokens/s | Q4 量化 |

## 八、自启动配置

### 8.1 Ollama 开机自启

Ollama 安装后默认注册为 launchd 服务，开机自动启动。

如需手动管理：
```bash
# 查看服务状态
launchctl list | grep ollama

# 停止服务
launchctl stop com.ollama.server

# 启动服务
launchctl start com.ollama.server
```

### 8.2 创建快捷启动脚本

在 MacBook 上创建 `start-model.sh`：

```bash
#!/bin/bash
export OLLAMA_HOST=0.0.0.0:11434
export OLLAMA_NUM_GPU=99
export OLLAMA_NUM_CTX=8192
export OLLAMA_KEEP_ALIVE=24h

echo "Starting Ollama server..."
ollama serve &
sleep 3

echo "Loading Qwen2.5-VL model..."
ollama run qwen2.5vl:7b --keepalive 24h "test" > /dev/null 2>&1

echo "Model ready! API endpoint: http://$(ifconfig en0 | grep 'inet ' | awk '{print $2}'):11434/v1"
```

## 九、故障排查

### 9.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| Windows 连接超时 | 防火墙阻止 | 检查 macOS 防火墙设置，添加 Ollama 例外 |
| 模型加载失败 | 内存不足 | 关闭其他应用释放内存，或使用更小的量化版本 |
| 推理速度慢 | 未启用 Metal | 确认 `OLLAMA_NUM_GPU=99` 或 llama.cpp 编译时 `LLAMA_METAL=1` |
| 中文乱码 | 编码问题 | 确保请求/响应 Content-Type 包含 `charset=utf-8` |
| Ollama 端口拒绝连接 | 未绑定 0.0.0.0 | 设置 `OLLAMA_HOST=0.0.0.0:11434` |
| API 返回空结果 | 模型未预加载 | 先执行 `ollama run qwen2.5vl:7b` 预热模型 |

## 十、本地模型 vs 云端 API 对比

| 对比维度 | 本地部署 (MacBook) | 云端 API (DashScope) |
|---------|-------------------|---------------------|
| **延迟** | ~1-2s TTFT | ~2-5s TTFT (含网络) |
| **成本** | 硬件一次性投入 | 按 Token 计费 |
| **隐私** | 数据不出内网 | 数据发送至云端 |
| **可用性** | 依赖本机运行 | 99.9% SLA |
| **并发** | 受硬件限制 (2-4 并发) | 高并发支持 |
| **模型更新** | 需手动更新 | 自动版本管理 |
| **适用场景** | 开发调试、隐私敏感 | 生产环境、高并发 |
