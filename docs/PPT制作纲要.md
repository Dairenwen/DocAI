## 第10页：8阶段智能填表流水线
**设计要点**：
- 竖向流程图（8阶段）：
  1. 阶段0：统一数据契约（6个核心数据结构）
  2. 阶段1：源文档结构化抽取（qwen-turbo）
  3. 阶段2：字段标准化 + 别名词典
  4. 阶段3：模板槽位识别（代码规则，不用模型）
  5. 阶段4：规则候选召回（Top-5）
  6. 阶段5：LLM难例判定（qwen-plus，仅限难例）
  7. 阶段6：置信度融合 + 拒填策略
  8. 阶段7：模板写回 + 二次校验
- 每阶段标注是否调用模型

## 第11页：关键创新点
**设计要点**：
- 三大创新用卡片式布局：
  
  **创新一：规则+AI混合决策架构**
  - 优先走规则召回，减少LLM调用成本
  - 仅当Top-1与Top-2分差<0.15或Top-1<0.75时调用LLM
  - 低置信度(<0.70)主动拒填，避免错填

  **创新二：全链路可追溯审计**
  - 每个填入值记录来源文档、原文证据、决策原因
  - 支持答辩时直接展示决策过程

  **创新三：多格式统一抽取框架**
  - docx通过POI段落+表格解析
  - xlsx逐sheet逐行解析
  - txt/md直接文本处理
  - 统一输出ExtractedField结构

## 第12页：准确率保障机制
**设计要点**：
- 10条硬规则列表（精简版）：
  1. 抽取与填写彻底分离
  2. 模型只做选择题，不做开放式生成
  3. 所有字段必须带原文证据
  4. 低置信度允许拒填，不允许乱填
  5. 写回后必须二次校验
- 右侧配置信度分级图：
  ```
  ≥ 0.85  →  ✅ 直接填写
  0.70~0.85 → ⚠️ 填写+标记复核
  < 0.70   → ❌ 拒填保空白
  ```

## PPT设计风格建议
- **主色调**：紫蓝渐变（#4F46E5 → #7C3AED），与系统UI一致
- **字体**：标题用黑体/Impact，正文用微软雅黑/Inter
- **图标**：使用扁平化图标，保持视觉统一
- **动画**：适度使用淡入/滑入动画，避免过度
- **每页文字**：不超过50字，重要数字放大突出
