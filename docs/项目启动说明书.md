# DocAI 项目启动说明书

## 一、环境要求

### 1.1 基础环境
| 软件 | 最低版本 | 说明 |
|------|---------|------|
| JDK | 17+ | 后端运行环境 |
| Maven | 3.8+ | 后端构建工具 |
| Node.js | 18+ | 前端构建环境 |
| npm | 9+ | 前端包管理器 |
| Docker | 24+ | 容器化中间件 |
| Docker Compose | v2+ | 多容器编排 |

### 1.2 硬件要求
| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 10GB | 20GB+ |
| 网络 | 可访问阿里云API | 低延迟网络 |

### 1.3 端口需求
| 端口 | 服务 | 说明 |
|------|------|------|
| 8080 | Nginx | 前端访问入口 |
| 18080 | Gateway | API网关 |
| 9001 | user-service | 用户服务 |
| 9002 | ai-service | AI服务 |
| 9003 | file-service | 文件服务 |
| 3306 | MySQL | 数据库 |
| 6379 | Redis | 缓存 |
| 8848 | Nacos | 配置中心 |

## 二、环境变量配置

在启动前，请配置以下环境变量（必选项用 * 标注）：

### 2.1 AI模型配置（*必须）
```bash
# 阿里云DashScope API Key（必须配置才能使用AI功能）
export DOC_DASHSCOPE_API_KEY=your-dashscope-api-key

# DeepSeek API Key（可选，作为备用模型）
export DOC_DEEPSEEK_API_KEY=your-deepseek-api-key
```

获取方式：
1. 登录 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 创建或使用已有的 API Key
3. 确保有 qwen-plus 模型的调用权限

### 2.2 邮件服务配置（可选）
```bash
export DOC_SMTP_HOST=smtp.qq.com
export DOC_SMTP_PORT=587
export DOC_SMTP_USER=your-email@qq.com
export DOC_SMTP_AUTH_CODE=your-smtp-auth-code
```

### 2.3 阿里云OSS配置（可选，不配置则使用本地存储）
```bash
export DOC_OSS_ENDPOINT=oss-cn-beijing.aliyuncs.com
export DOC_OSS_BUCKET=your-bucket-name
export DOC_OSS_ACCESS_KEY_ID=your-access-key-id
export DOC_OSS_ACCESS_KEY_SECRET=your-access-key-secret
```

## 三、Windows 平台启动

### 3.1 一键启动
```powershell
# 1. 打开 PowerShell（管理员模式）
# 2. 进入项目目录
cd F:\DocAI\docai-pro

# 3. 设置API Key环境变量
$env:DOC_DASHSCOPE_API_KEY = "your-api-key-here"

# 4. 一键启动（包含构建、部署、启动全流程）
.\start-lite-windows.ps1
```

### 3.2 启动流程说明
脚本会自动执行以下6个步骤：
1. **检查依赖** — 验证 Docker、Maven、npm、Java 是否安装
2. **构建后端** — `mvn -DskipTests package` 编译所有微服务
3. **构建前端** — `npm install && npm run build` 编译Vue前端
4. **同步资源** — 将前端构建产物复制到Nginx目录
5. **启动中间件** — Docker Compose 启动 MySQL、Redis、Nacos、Nginx
6. **启动服务** — 依次启动 user-service、file-service、ai-service、gateway-service

### 3.3 停止服务
```powershell
.\stop-lite-windows.ps1
```

## 四、Linux 平台启动

### 4.1 一键启动
```bash
# 1. 进入项目目录
cd /path/to/DocAI/docai-pro

# 2. 设置API Key
export DOC_DASHSCOPE_API_KEY="your-api-key-here"

# 3. 给脚本添加执行权限
chmod +x start-lite.sh stop-lite.sh

# 4. 一键启动
./start-lite.sh
```

### 4.2 后台运行
```bash
# 使用 nohup 后台运行
nohup ./start-lite.sh > startup.log 2>&1 &

# 查看启动日志
tail -f logs/ai-service.log
```

### 4.3 停止服务
```bash
./stop-lite.sh
```

### 4.4 服务器部署注意事项
- 确保服务器防火墙开放 8080 端口（前端访问）
- 如需外网访问，可配置 `PUBLIC_HOST` 环境变量为服务器公网IP
- 建议使用 systemd 或 supervisor 管理Java进程

## 五、首次启动后数据库初始化

Docker Compose 会在首次启动时自动执行数据库初始化：
1. 创建 `docai` 业务数据库
2. 创建 `docai_nacos` 配置中心数据库
3. 创建数据库用户 `drw`
4. 初始化所有业务表结构（12张核心表）

如需手动初始化，执行：
```bash
# 进入MySQL容器
docker exec -it docai-mysql mysql -uroot -pDairenwen1092

# 执行建表脚本
source /opt/sql/docai_schema.sql;
```

## 六、访问系统

启动完成后，浏览器访问：

| 入口 | 地址 | 说明 |
|------|------|------|
| 系统首页 | http://localhost:8080 | 前端应用入口 |
| API网关 | http://localhost:18080 | 后端API入口 |
| Nacos控制台 | http://localhost:8848/nacos | 配置管理（用户：nacos/nacos） |

### 6.1 注册与登录
1. 打开 http://localhost:8080
2. 点击「注册」创建新账号（用户名 + 密码）
3. 登录后进入工作台

### 6.2 核心流程
1. **文档管理** → 上传源文档（支持 .docx / .xlsx / .txt / .md 四种格式）
2. 系统自动提取文档内的结构化字段（AI + 规则双引擎），通过轻量级状态轮询实时显示处理进度
3. 支持文件预览（原文件预览）、下载、删除操作
4. 支持批量勾选后一键下载已选 / 删除已选文档
5. **AI对话** → 关联已提取文档进行智能问答、文档翻译、润色优化等
6. **智能填表** → 上传模板文件（Word/Excel表格模板）
7. 选择一个或多个数据源文档 → 点击「开始填充」
8. 等待AI处理 → 下载填充结果

## 七、常见问题

### Q1: Docker容器启动失败
```bash
# 检查容器状态
docker compose -f deploy/docker-compose-mid-windows.yml ps
# 查看日志
docker compose -f deploy/docker-compose-mid-windows.yml logs docai-mysql
```

### Q2: AI服务报错"LLM调用失败"
- 检查 `DOC_DASHSCOPE_API_KEY` 是否正确配置
- 检查网络是否可以访问 `dashscope.aliyuncs.com`
- 查看日志：`logs/ai-service.log`

### Q3: 文件上传失败
- 检查文件大小是否超过限制（默认100MB）
- 确认文件格式是否支持（仅支持 .docx / .xlsx / .txt / .md 四种格式）
- 查看日志：`logs/ai-service.log`

### Q4: 前端页面白屏
- 确认Nginx容器正常运行
- 检查前端构建产物是否正确复制到 `deploy/nginx/web/dist/`
- 重新执行前端构建：`cd docai-frontend && npm run build`

### Q5: 数据库连接失败
- 确认MySQL容器已启动且端口3306可访问
- 首次启动时MySQL初始化需要30-60秒
- 检查数据库用户密码是否正确
